Publica tu trabajo en internet

Vas a subir tu actividad a GitHub y a publicarla con GitHub Pages para que tenga una URL pública. Entregas esa URL más el enlace del repositorio.

1 · Organiza tu repositorio

Una carpeta por reto y una portada que enlace a las tres:

tu-repositorio/
├── index.html      (portada con enlaces a los 3 retos)
├── bem/
│   └── index.html  (+ tus .css)
├── oocss/
│   └── index.html  (+ tus .css)
└── smacss/
    ├── index.html
    └── css/        (base, layout, module, state, theme)

2 · Súbelo con git

git init
git add .
git commit -m "Actividad metodologías CSS"
git branch -M main
git remote add origin https://github.com/TU_USUARIO/TU_REPO.git
git push -u origin main
Haz varios commits con mensajes claros mientras avanzas (uno por reto, por ejemplo). También cuenta en la evaluación.

3 · Activa GitHub Pages

  • En tu repo, entra en Settings → Pages.
  • En "Build and deployment", elige Deploy from a branch.
  • Selecciona la rama main y la carpeta / (root). Guarda.
  • Espera 1-2 minutos y abre la URL pública que aparece arriba.
  • Comprueba que se ven los tres retos desde tu portada.

4 · Qué entregar

Pega en la entrega:

🔗 URL de GitHub Pages (tu sitio publicado).
🔗 URL del repositorio en GitHub.

Cómo se evalúa

CriterioQué miramos
BEM correctoSolo clases, convención de nombres, bloque/elemento/modificador bien usados.
OOCSS correctoEstructura separada de skin, objetos reutilizables, sin reglas atadas al contenedor.
SMACSS correctoCSS repartido en las 5 categorías y estados funcionando con JS.
Git y entregaRepo ordenado, commits claros y GitHub Pages publicado y funcionando.
Una metodología por reto. No mezcles BEM, OOCSS y SMACSS en el mismo reto: la gracia es practicar cada lógica por separado.
